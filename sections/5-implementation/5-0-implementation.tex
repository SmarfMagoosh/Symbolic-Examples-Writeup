\section{Implementation}
\input{sections/figures/test2-eq}
We implement our property-based testing approach
in \ourDSL, an eDSL for Haskell.
The eDSL provides functions \hskin{testN}
(for $N$ from 1 to 6)
to run a function with $N$ arguments
on $N$ expression specified by symbolic expressions,
and check if the value returned can be validated by another symbolic expressions.
(Examples of this are shown earlier in the
paper, in Figure~\ref{fig:zip-test}, for example.)

We now consider how this works,
 considering \hskin{test2} in particular
(the other \hskin{testN} functions are similar.)
The type of \hskin{test2} is shown in Figure~\ref{fig:test2-eq}.
This type involves GADTs and TypeFamilies,
two advanced Haskell type features .
(Note \ourDSL can be \textit{used} without an understanding of GADTs, type families,
or the details of how the \hskin{testN} works--
this section is providing an ``under the hood'' look at how the eDSL works.)
We also use QuasiQuoters to allow specifying symbolic expressions.
We first discuss our use of Quasiquoters.
Then we discuss
GADTs and Type Families~\cite{sulzmann2007system},
and elaborate on how they are used in \ourDSL.
