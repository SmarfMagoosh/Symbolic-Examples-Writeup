\subsection{Generation Rules} \label{subsec:generation-rules}

\input{sections/figures/generation-rules}

\par 
We now consider generating random input values from symbolic expressions. 
To do this, we define a relation $\inp \lab \gens x, \zout$. 
Figure~\ref{fig:generation-rules}
shows the generation rules.

\mypara{Lists Constructors}
\RuleEmpty, \RuleCons, and \RuleConcat
structurally match on a symbolic list,
recursively generating a concrete list's subcomponents,
and then appropriately composing those subcomponents together.
\RuleEmpty is the simplest of these,
simply saying that an empty list
can only generate an empty list
($\inp \nil \gens \nil, \zout$.)
\RuleCons generates a single element
for the head of the list
and generates the tail of the list,
and composes them with \cons.
Finally, \RuleConcat generates
two separate lists,
and concatenates them together with \concat.


\mypara{Literals}
\RuleLiteral matches on a specific literal--
in a similar fashion to \RuleEmpty,
this rule simply straightforwardly says that a literal 
can only generate exactly that literal.

\mypara{Assignment and Variables}
\RuleAssign
handles binding of a new label,
$(\lab \Longleftarrow p(args, \lab)) \mapsto e$.
To do this,
it repeatedly generates arbitrary values of the appropriate type
\bill{how do we know appropriate type?},
until it finds a value that satisfies the constraint $p(args, \lab)$.
Once such a value is found,
the label is bound and returned in a new environment,
$\outw{\lab}{x}$.
Given a label appearing in the expression,
\RuleVariable then allows looking up that label in the environment,
and generating the mapped-to value.

\mypara{Union}
\RuleUnionLeft and \RuleUnionRight
handle the $\|_\lab$ operator.
Note that which rule applies
is based on the (previously randomly chosen)
value of $\lab$,
hence getting us an (indirectly) non-deterministic choice.

\mypara{Kleene Closure}
\RuleStar handles kleene closures, $r*_\lab$.
To do this, it looks up $\envget{\lab} = n$ in the environment,
and then recursive applies \gens to generate $n$ lists that match $r$,
i.e.
$\forall 1 \leq i \leq n . (\env_{i - 1}, \envput{\pathvar}{\lab}{i}) r \gens xs_i, \env_i$.
Note that (1) the environment is threaded through all these calls,
to ensure that we record all variable bindings performed within each regular expression
and (2) as covered in our description of the path above,
this is the one place in the rules where we \textit{modify} the path,
by, on each iteration, setting the label \lab to point to the current iteration count $i$.

\mypara{Apply}
\RuleApply allows data constructors to be applied to symbolic expression arguments.
To do this, the Apply constructor must have a function, 
and list of symbolic expressions representing the arguments such that when the arguments are used to generate values, 
the functions can be applied to those values in order to produce an output.
